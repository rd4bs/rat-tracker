Gym Tracker PWA Requirements
Last updated: 2026-06-21

Project Purpose
- Single-user workout-tracking PWA for desktop and mobile.
- Store workout plans, workout results, exercise data, notes, and muscle-density information locally.
- Preserve local Dexie / IndexedDB data unless an explicit migration is required.
- Preserve the desktop layout while improving mobile usability.
- Support use from desktop and phone during local-network testing.

Technology Requirements
- Vite
- React
- TypeScript
- Dexie / IndexedDB
- VS Code on Windows

Development Rules
- Inspect existing types and components before changing architecture.
- Prefer small, testable changes.
- Do not rename files or move folders unless required.
- Preserve Dexie data compatibility.
- Do not infer workout status from exercises or sets.
- Workout status changes should come from explicit user actions.
- Run lint/build after code changes and report any errors.
- Commit stable checkpoints when requested so the project can be rolled back for debugging.
- Do not mark a feature complete until desktop and phone behavior has been confirmed by the user.

PWA Requirements
- App must have a valid manifest.
- App must have app icons for install surfaces.
- App must register a service worker in a secure context.
- App shell must be available offline after successful service worker installation.
- App must show useful status for online/offline and PWA readiness.
- App must provide a way to apply available app updates.
- App must preserve IndexedDB workout data across reloads, browser restarts, and installed-app launches.
- App must support local backup/export of user data.
- App must support local import/restore of a Gym Tracker backup.
- App must handle IndexedDB save/load failures gracefully.
- Full service worker and install testing requires HTTPS or localhost. LAN HTTP can test the app UI, but browsers usually block service workers on http://192.168.x.x.
